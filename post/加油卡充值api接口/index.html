加油卡充值API接口

接口调用前，请先联系客服人员，索取地址，开通对应充值权限，分配appid和key，以及配置ip白名单。加油卡完成网上充值后，需到所充值油卡当地加油站请工作 人员进行激活(圈存)操作，否则充值金额将无法生效。发票等相关问题请联系客服。

22:40-1:00 属于系统维护时间，此时间段内提交的申请，会延时至1:01 进行处理。如果在此时间段内进行充值，请耐心等待回调通知，不必进行大量查询。

充值请求提交

  • 接口地址： /api/charge
  • 请求方式： POST
  • 返回格式： json
  • 备注：如果充值过程中，遇到http网络状态异常或返回系统异常，请务必通过订单查询接口检测订单或联系客服，不要直接做失败处理，避免造成不必要的损失!!!

请求参数说明：

名称 类型 是否必填 说明
app_id string Y 系统分配的用户账号
out_order_no string Y 商户自定义订单号，注：每个商户在系统内只应拥有唯一一个订单号，且为40字符以内的字母数字组合
user_card_id string Y 充值加油卡卡号
amount string Y 充值金额
timestamp string Y 商户订单提交时间戳，时间误差应控制在上下五分钟以内
sign string Y 订单签名（签名方式详见下方签名说明）

充值结果通知

  • 通知方式： POST
  • 通知格式： json
  • 备注：结果通知接收后，请务必回复 “SUCCESS” , 如果不回复或回复不正确，则系统会以 15/30/60/120/240/480/960/1920/3840/7680 的间隔进行重试，直到正确返回或达到次数限制。
    注意：同样的通知可能会多次发送给商户系统。商户系统必须能够正确处理重复的通知。 推荐的做法是，当收到通知进行处理时，首先检查对应业务数据的状态，判断该通知是否已经处理过，如果没有处理过再进行处理，如果处理过直接返回结果成功。在对业务数据进行状态检查和处理之前，要采用数据锁进行并发控制，以避免函数重入造成的数据混乱。
    特别提醒：商户系统对于结果通知的内容一定要做签名验证,并校验返回的订单金额是否与商户侧的订单金额一致，防止数据泄漏导致出现“假通知”，造成资金损失。

订单查询

  • 接口地址： /api/select
  • 请求方式： POST
  • 返回格式： json
  • 备注：一般用于结果通知未能正常处理后的补救，需要注意的是，系统维护时间内提交成功后，等待时间会延长，最长等待时间约为100分钟，如100分钟后仍未收结果，且订单查询异常，请联系客服人员进行处理

签名方式

  • 提交数据

数据签名方式为提交数据除“sign”本身外的所有key-value键值对，将分配的md5key加入进去后，以key值进行排序，再将value进行拼接，最后再用进行md5转换为32位小写

举例：

充值提交数据：
app_id: AppID
out_order_no: OutOrderNo
user_card_id: 1234567890123456789
amount: 10000
timestamp: 1537425980

用户md5key: Md5Key

加入md5key后将key排序：
amount app_id md5key out_order_no timestamp user_card_id

以key排序后，生成待签名字符串为：
10000AppIDMd5KeyOutOrderNo15374259801234567890123456789

md5 生成32位小写签名字符串
6bc6fbeecc874533ca0d87e9964fe936

最终提交数据
{
  "amount":"10000",
  "app_id":"AppID",
  "out_order_no":"OutOrderNo",
  "user_card_id":"1234567890123456789",
  "timestamp":"1537425980",
  "sign":"6bc6fbeecc874533ca0d87e9964fe936"
}
  • 验证签名

订单签名需要将 分配的md5key加入到 [Data]中的 kv对，再以key排序，并连接value，然后用md5生成32位小写字符串与返回的sign进行比较。

需要注意的是，只有[Data]中的数据才会参与签名

举例：

提交结果后收到的回复数据：
{
  "code":0,
  "msg":"成功",
  "data":{
    "amount":"10000",
    "balance":"0",
    "cash":"12000",
    "credit":"24000",
    "order_no":"5ba9f91fe9f2122a4cc2561b",
    "out_order_no":"OutOrderNo",
    "product":"加油充值100元",
    "result_code":"SUCCESS",
    "result_msg":"SUCCESS",
    "timestamp":"1537866018",
    "user_card_id":"1234567890123456789"
  },
  "sign":"df9915b578e9090e516ebaa0e83f821e"
}

md5key : Md5Key

data中数据加入md5key后以key排序：
amount balance cash credit md5key order_no out_order_no product result_code result_msg timestamp user_card_id

拼接value字符串为：
1000001200024000Md5Key5ba9f91fe9f2122a4cc2561bOutOrderNo加油充值100元SUCCESSSUCCESS15378660181234567890123456789

md5转换后：
df9915b578e9090e516ebaa0e83f821e

与返回sign对比
一致，验签成功

常见问题

  • 20000 输入参数格式错误
    一般是由于发送的数据不是json格式所造成的，请检查所发送的数据格式，并确定Content-Type 为 application/json， 发送http方法为post

  • 20039 不合规范的订单号
    一般是由于字段大小写不一致所造成，请确认发送的json数据中 key值的大小写，如请不要将 app_id 写成 AppID, AppId, app_ID, app_Id等格式
